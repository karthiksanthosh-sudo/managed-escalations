# Managed Escalation Tracker

Static dashboard (GitHub Pages) + Google Sheets backend, auto-synced from Slack every hour via GitHub Actions.

## Architecture

```
GitHub Actions (cron: every hour)
    │
    ├─ sync.py ─┬─ Slack API (C0A8GNBCPRR)  →  parse threads
                ├─ Google Sheets API         →  upsert Escalation Tracker + Daily Summary
                └─ data.json                 →  commit to repo
                         │
                   GitHub Pages
                   index.html  ←  fetch('./data.json')  →  live dashboard
```

The HTML page re-fetches `data.json` every 5 minutes automatically.
GitHub Actions commits a fresh `data.json` every hour.

## Setup (one-time, ~15 minutes)

### 1. Fork / create the repo on GitHub

```bash
cd ~/Downloads/TAM/escalation-web
git init
git add .
git commit -m "Initial commit"
gh repo create escalation-tracker --public --push --source=.
```

Or create a new repo on GitHub and push manually.

### 2. Enable GitHub Pages

Repo → **Settings → Pages → Source: Deploy from branch → main → / (root)**

Your dashboard will be at: `https://<your-username>.github.io/escalation-tracker/`

### 3. Create the Slack App

1. https://api.slack.com/apps → **Create New App → From scratch**
2. Name: `Escalation Tracker Bot` | Workspace: Razorpay
3. **OAuth & Permissions → Bot Token Scopes**, add:
   - `channels:history`
   - `channels:read`
   - `users:read`
   - `groups:history` (if the channel is private)
4. **Install to Workspace → Allow**
5. Copy the **Bot OAuth Token** (`xoxb-...`)
6. Invite bot to the channel: `/invite @Escalation Tracker Bot`

### 4. Create Google Service Account

1. Go to https://console.cloud.google.com → select or create a project
2. **APIs & Services → Enable APIs → enable:**
   - Google Sheets API
   - Google Drive API
3. **IAM & Admin → Service Accounts → Create Service Account**
   - Name: `escalation-tracker`
   - Role: Editor (or create a custom role with Sheets read/write)
4. **Keys → Add Key → JSON** → download the file
5. Copy the entire JSON content

### 5. Create the Google Sheet

1. https://sheets.new → create blank sheet
2. Share it with the service account email (from the JSON: `client_email` field) — give **Editor** access
3. Copy the spreadsheet ID from the URL:
   `https://docs.google.com/spreadsheets/d/SPREADSHEET_ID/edit`

### 6. Add GitHub Secrets

Repo → **Settings → Secrets and variables → Actions → New repository secret**

| Secret name | Value |
|------------|-------|
| `SLACK_BOT_TOKEN` | `xoxb-...` |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | Entire JSON from step 4 (paste as-is) |
| `GOOGLE_SHEET_ID` | Spreadsheet ID from step 5 |
| `LEADERSHIP_SLACK_USER_IDS` | Comma-separated Slack user IDs for SHK/Harshil/Khilan (optional but recommended) |

**How to find a Slack User ID:** Open Slack → click person's profile → three-dot menu → **Copy member ID**

### 7. Run initial backfill

GitHub → **Actions → Sync Escalation Data → Run workflow**
- Set `full_backfill` to `true` for the first run (loads 365 days of history)

After ~5–10 minutes, the workflow commits `data.json` and your dashboard is live.

## Google Sheets structure

Three sheets are created automatically:

**Escalation Tracker** — one row per thread:
Thread TS, Merchant Name, Who Raised, Escalation Manager, Created (IST),
Escalation Context, Current Status, Resolution Status, TTR (Business Hours),
SLA Breach (8h), Escalation Month, Merchant Unblocked, Merchant Confirmation,
Mx Confirm Timestamp, BL Approval, Escalation Type, IM Involved, Thread URL,
Total Replies, Last Activity (IST), Last Updated By, Resolution Timestamp,
Open Days, Week #, Quarter

**Daily Summary** — aggregated per day:
Date, Total, Open, Resolved, SLA Breaches, SLA %, Avg TTR (h), Leadership, AM, IM Involved

**SLA Dashboard** — auto-populated monthly breakdown

---

## SLA logic

- **Threshold:** 8 business hours
- **Business hours:** Mon–Sat, 10:00–19:00 IST
- **Breach = YES** if time from thread creation to first resolution keyword > 8 BH
- **Breach = Open** if thread is still unresolved

## Dashboard features

- KPI cards: Total, Open, Resolved, SLA %, Leadership, AM, IM, Avg TTR, Mx Pending
- Charts: Monthly bar (total/resolved/breached), SLA donut, Escalation type donut
- Filter by: Month, Escalation Type, Status, SLA status, free-text search
- Sortable table on any column
- Pagination (25 rows/page)
- Auto-refresh every 5 minutes (re-fetches data.json)
- Manual Refresh button

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| Dashboard shows "Failed to load data" | Check that `data.json` exists in the repo root; run the workflow manually |
| GitHub Actions failing | Check Secrets are set correctly; view the Actions log for the specific error |
| Slack `channel_not_found` | Invite the bot: `/invite @Escalation Tracker Bot` |
| Google Sheets auth error | Confirm service account has Editor access on the sheet; check JSON is pasted correctly |
| Merchant name shows "Unknown" | Parent message has no extractable name pattern — edit the Sheets row manually |
| Old threads not appearing | Run workflow with `full_backfill=true` |
| Hourly sync not running | Check Actions → Sync Escalation Data is enabled (not disabled) |
